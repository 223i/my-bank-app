# My Bank Front App

Веб-приложение с клиентским HTML-интерфейсом для банковской системы.

## 📋 Обзор

Front UI — это веб-приложение с клиентским HTML-интерфейсом, которое предоставляет пользователям доступ к банковским операциям через единый интерфейс.

### Основные функции
- **Аутентификация/авторизация** через OAuth 2.0 Authorization Code Flow
- **Управление профилем** - редактирование персональных данных
- **Финансовые операции** - пополнение и снятие средств
- **Переводы** - переводы денег между аккаунтами
- **JWT токенизация** - безопасная работа с микросервисами

### Структура интерфейса
- **Блок данных аккаунта** - управление персональными данными
- **Блок операций с деньгами** - пополнение и снятие средств
- **Блок переводов** - переводы между аккаунтами

## 🌐 Порты и эндпоинты

### Порт
- **8083** - основной порт приложения

### Основные страницы
- `/` - Главная страница (редирект на /account)
- `/account` - Управление аккаунтом
- `/login` - Страница входа (OAuth 2.0)

### API Эндпоинты (через Gateway)
- `GET /api/accounts/me` - Получение данных аккаунта
- `PUT /api/accounts/me` - Обновление данных аккаунта
- `GET /api/accounts/others` - Получение списка других пользователей
- `POST /api/cash/process` - Операции с деньгами
- `POST /api/transfer` - Переводы между счетами

## 🔐 Безопасность

### OAuth 2.0 Configuration
- **Authorization Code Flow** для аутентификации пользователей
- **JWT Bearer Token** для доступа к микросервисам
- **Token Storage** - хранение токена в сессии
- **Automatic Refresh** - обновление токена при необходимости

### JWT Claims
- **preferred_username** - логин пользователя
- **scope** - права доступа к сервисам
- **roles** - роли пользователя

### Права доступа
- **ROLE_ACCOUNTS_USER** - доступ к управлению аккаунтом
- **ROLE_CASH_USER** - доступ к операциям с деньгами
- **ROLE_TRANSFER_USER** - доступ к переводам

## 🛠️ Конфигурация

### Переменные окружения (.env)

```env
# Gateway
GATEWAY_URL=http://localhost:8080

# OAuth 2.0 / Keycloak
KEYCLOAK_ISSUER_URI=http://localhost:8080/realms/my-bank-app
OAUTH2_CLIENT_ID=my-bank-front
OAUTH2_CLIENT_SECRET=your-client-secret

# Приложение
SERVER_PORT=8083
SPRING_PROFILES_ACTIVE=docker
```

### Структура конфигурации
```yaml
# application.yaml
spring:
  application:
    name: my-bank-front-app
  security:
    oauth2:
      client:
        registration:
          keycloak:
            client-id: ${OAUTH2_CLIENT_ID}
            client-secret: ${OAUTH2_CLIENT_SECRET}
            authorization-grant-type: authorization_code
            redirect-uri: http://localhost:8083/login/oauth2/code/keycloak
            scope: openid,profile,email
        provider:
          keycloak:
            issuer-uri: ${KEYCLOAK_ISSUER_URI}
      resourceserver:
        jwt:
          issuer-uri: ${KEYCLOAK_ISSUER_URI}
```

## 🎨 Интерфейс пользователя

### Блок данных аккаунта пользователя

Состоит из:
- **Фамилия и имя** - поле с возможностью редактирования
- **Дата рождения** - поле с возможностью редактирования
- **Текущая сумма на счету** - отображение без возможности редактирования
- **Кнопка «Сохранить изменения»** - сохранение внесенных изменений

**Валидация:**
- Все поля должны быть заполнены
- Возраст должен быть старше 18 лет
- При сохранении невалидных данных отображается ошибка

### Блок внесения и снятия виртуальных денег

Состоит из:
- **Поле ввода суммы** - обязательное для заполнения
- **Кнопка «Положить»** - пополнение счета
- **Кнопка «Снять»** - снятие средств со счета

**Валидация:**
- Сумма должна быть положительным числом
- Если сумма снятия больше баланса - отображается ошибка

### Блок перевода денег на счёт другого аккаунта

Состоит из:
- **Поле выбора аккаунта** - выпадающий список пользователей (обязательно)
- **Поле ввода суммы перевода** - обязательное для заполнения
- **Кнопка «Перевести»** - выполнение перевода

**Валидация:**
- Получатель должен быть выбран
- Сумма перевода должна быть положительной
- Если сумма перевода больше баланса отправителя - отображается ошибка

## 🔄 Пользовательские сценарии

### Управление профилем
1. Пользователь входит в систему
2. Видит свои данные (имя, дата рождения, баланс)
3. Редактирует имя и/или дату рождения
4. Нажимает «Сохранить изменения»
5. Система валидирует данные и сохраняет изменения

### Пополнение счета
1. Пользователь вводит сумму для пополнения
2. Нажимает «Положить»
3. Система выполняет операцию
4. Отображает успешное сообщение и обновляет баланс

### Снятие средств
1. Пользователь вводит сумму для снятия
2. Нажимает «Снять»
3. Система проверяет достаточность средств
4. При успехе выполняет операцию и обновляет баланс
5. При недостатке средств отображает ошибку

### Перевод денег
1. Пользователь выбирает получателя из списка
2. Вводит сумму перевода
3. Нажимает «Перевести»
4. Система проверяет валидность и достаточность средств
5. При успехе выполняет перевод и обновляет баланс
6. При ошибках отображает соответствующее сообщение

## 📊 Компоненты

Утилита для работы с JWT токенами:
- Извлечение логина пользователя
- Получение ролей и прав доступа
- Проверка срока действия токена

Сервис для взаимодействия с микросервисами:
- Получение данных аккаунта
- Выполнение финансовых операций

Основной контроллер приложения:
- Отображение главной страницы
- Обработка POST запросов
- Управление состоянием приложения
